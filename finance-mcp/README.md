# Lumen Finance MCP Server

A Model Context Protocol (MCP) server that exposes the finance system of a
fictional post-seed B2B SaaS, **Lumen Inc.**, operating through 6 legal
entities in 6 currencies (USD, GBP, EUR, BRL, ARS, INR).

Any MCP-compatible client (Claude, Cowork, etc.) can connect and operate it
in natural language: "show me the consolidated P&L for May in EUR",
"what's our cash runway", "give me the AR aging".

## Tools

| Tool | What it returns |
|------|-----------------|
| `list_entities` | Legal entities, country, functional currency |
| `get_pnl` | Consolidated P&L for a period, in any reporting currency |
| `get_balance_sheet` | Consolidated balance sheet at the latest close |
| `get_ar_aging` | Accounts receivable aging by bucket |
| `get_cash_position` | Cash by entity and consolidated |

Every tool takes a `report_currency` and most take an optional `entity_id`,
so the same tool serves both consolidated and single-entity views.

## Design decisions (the "why")

- **Multi-currency consolidation uses period-close FX, not live rates.**
  Rates live in `data/fx_rates.csv`, frozen per period. A closed period's
  P&L must not change over time. Pulling live FX would silently restate
  prior months. This mirrors real consolidation.
- **Double-entry integrity.** The balance sheet balances per entity
  (Assets = Liabilities + Equity), with retained earnings as the plug, the
  standard accounting practice. For a post-seed company this surfaces as an
  accumulated deficit, which is the honest picture.
- **Separation of concerns.** A thin data layer loads the CSVs; pure FX
  helpers handle conversion; each tool composes them. Easy to test, easy to
  point at a different data source (e.g. a real ERP export or connector).
- **Inputs are validated.** Unknown currencies or periods return a clear
  error listing valid options, instead of failing silently.

## Data model (`data/`)

- `entities.csv` — the 6 legal entities and their currency
- `chart_of_accounts.csv` — account codes, names, types
- `fx_rates.csv` — period-close rates (local units per 1 USD)
- `pnl_activity.csv` — monthly P&L activity per entity (local currency)
- `balance_sheet.csv` — balances per entity (local currency) for **two periods**
  (prior + latest close), so the financial statements articulate and the indirect
  cash flow foots. Consumers must filter by period (default = latest close)
- `ar_invoices.csv` — open/paid invoices for aging
- `ap_invoices.csv` — vendor bills (payables): open/paid, due dates, currency
- `tax_obligations.csv` — tax obligations per entity/jurisdiction (VAT, income, withholding), with due dates and status
- `budget.csv` — the operating plan per entity/period/account, in USD (a deliberate plan, not actuals + noise)

These extra series (budget, AP, tax) feed the `cfo-office/` agents (FP&A
budget-vs-actual, and the Administration sub-agents AR/AP/Tax). Data is
synthetic and generated by `generate_data.py` (reproducible, seeded).

## Run it

```bash
pip install -r requirements.txt
python generate_data.py        # regenerates data/ (optional, already included)
python server.py               # starts the MCP server (stdio)
```

To explore the tools interactively without a full client:

```bash
pip install mcp
mcp dev server.py              # opens the MCP Inspector in the browser
```

## Notes

Synthetic data; the architecture and accounting logic are built to point at
production data.
